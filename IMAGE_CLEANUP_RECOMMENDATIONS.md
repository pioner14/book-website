# Рекомендации по очистке дубликатов изображений

## Обнаруженные проблемы:

1. **Дублирующиеся подпапки**:
   - Каждая глава имеет подпапку с именем вида "N./" (например, "1./", "2./")
   - Эти подпапки содержат дубликаты изображений из основной папки главы
   - Пример: `/public/chapter-images/1/` и `/public/chapter-images/1/1./`

2. **Дублирующиеся изображения с разными расширениями**:
   - Одинаковые изображения с разными расширениями (например, .jpg и .jpeg)
   - Используется только один из форматов, другой является дубликатом

3. **Неподходящие файлы**:
   - Файлы .docx в папках изображений (например, 1_.docx, 1./1_.docx)

## Рекомендации по очистке:

### 1. Удаление дублирующих подпапок:
Следует удалить все подпапки вида "N./" для каждой главы (1./, 2./ и т.д.), т.к. они содержат дубликаты изображений.

### 3. Сравнение с существующими файлами:
В директориях также есть дубликаты с другими расширениями:
- Например, в главе 1: 97EB435B-AAE5-4CF1-81F2-3A0BC7B93A65.jpeg и 97EB435B-AAE5-4CF1-81F2-3A0BC7B93A65.jpg (дубликат)

### 4. План очистки:
1. Удалить все подпапки вида "N./" в каждой главе
2. Удалить файлы .docx из папок изображений
3. Сравнить используемые изображения с файлами на диске и удалить дубликаты с другими расширениями
4. Оставить только те изображения, которые используются в MDX файлах

### 5. Проверка после очистки:
- Убедиться, что после очистки все изображения, используемые в MDX файлах, по-прежнему доступны
- Проверить отображение изображений на страницах глав
- Запустить сборку проекта для проверки отсутствия ошибок

### 6. Рекомендации по дальнейшему управлению:
- Внедрить процесс проверки при добавлении новых изображений
- Обновить инструкцию по добавлению иллюстраций с упоминанием необходимости избегать дубликатов
- Рассмотреть возможность использования CDN или системы оптимизации изображений для избежания дубликатов в будущем
